# buja-map: 자산관리 대시보드

스페이스X, 미국 M2 유동성, 나스닥 지수의 상관관계를 매일 업데이트하여 보여주는 웹 대시보드.

## 1. 목적

- 미국 M2 통화량(유동성)과 나스닥 지수, 그리고 스페이스X 관련 지표(밸류에이션/이벤트) 간의 관계를 한눈에 추적
- "유동성 확장 → 위험자산(나스닥) 상승 → 비상장 고성장주(스페이스X) 밸류에이션 상승" 가설을 시각적으로 검증
- 매일 자동 갱신되는 개인용 자산관리 참고 대시보드

## 2. 추적 지표

| 지표 | 설명 | 데이터 소스(안) | 갱신 주기 |
|---|---|---|---|
| 미국 M2 유동성 | M2SL (Seasonally Adjusted) | FRED API (`M2SL`) | 주간(FRED 발표 기준), 매일 폴링 |
| 나스닥 지수 | NASDAQ Composite (IXIC) 또는 QQQ | Yahoo Finance / Alpha Vantage / Stooq | 매일(장 마감 기준) |
| 스페이스X 지표 | 최근 라운드 기준 추정 밸류에이션, 직원 거래(tender offer) 가격, 관련 뉴스/이벤트 타임라인 | 공개 보도자료·뉴스 스크래핑(수동 입력 또는 반자동), 비상장이라 실시간 시세 없음 | 이벤트 발생 시 + 월간 리뷰 |

> 스페이스X는 비상장기업이라 일별 시세가 없음 — 라운드/텐더오퍼 밸류에이션을 계단식(step function)으로 표시하고, M2·나스닥과는 "동일 시점 비교"보다는 "국면(regime) 비교"로 해석하는 것이 현실적.

## 3. 상관관계 분석

- **M2 vs 나스닥**: 롤링 상관계수(예: 90일/180일 윈도우), M2 YoY% 증가율과 나스닥 수익률의 리드-래그(lead-lag) 분석 (M2가 선행지표인지 확인, 통상 3~12개월 리드 가설)
- **나스닥 vs 스페이스X 밸류에이션**: 라운드 시점의 나스닥 수준/추세와 밸류에이션 배수(revenue multiple 등) 비교
- **3자 종합 뷰**: 정규화(z-score 또는 indexed to 100)한 3개 라인을 하나의 타임시리즈 차트에 겹쳐서 표시

## 4. 아키텍처

```
[데이터 수집 (일 1회 배치)]
   FRED API ──┐
   Yahoo/Stooq ─┼─▶ 수집 스크립트 (Python/Node, cron) ─▶ DB/JSON (SQLite or 정적 JSON)
   뉴스/수동입력 ─┘

[백엔드/API]
   정적 JSON 서빙 또는 경량 API (FastAPI/Express)

[프론트엔드]
   대시보드 (Next.js/React + 차트 라이브러리)
   - 3-라인 타임시리즈 (정규화 지수)
   - 롤링 상관계수 차트
   - 스페이스X 이벤트 타임라인 마커
   - 최근 값 요약 카드 (오늘 M2, 나스닥, 최신 SpaceX 밸류에이션)

[스케줄링]
   매일 1회 배치 (예: GitHub Actions cron 또는 서버 cron) → 데이터 수집 → JSON 업데이트 → 배포
```

## 5. 데이터 소스 상세

- **FRED (Federal Reserve Economic Data)**: `https://fred.stlouisfed.org/series/M2SL` — 무료 API, API 키 필요
- **나스닥 지수**: Stooq(무료, 키 불필요) 또는 Alpha Vantage(무료 티어) 또는 Yahoo Finance 비공식 API
- **스페이스X**: 공개 보도(Bloomberg, Reuters, TechCrunch 등)에서 라운드/밸류에이션 뉴스 수동 큐레이션 → JSON 데이터 파일로 관리

## 7. 오픈 이슈

- 스페이스X는 실시간 시세가 없어 "매일 업데이트"라는 목표와 어긋남 → 스페이스X는 이벤트 기반 계단식 데이터로, M2·나스닥만 진짜 일일 갱신 대상으로 명확히 구분 필요
- M2는 실제로는 월간/2주 단위 발표(주간 데이터 지연 있음) → "매일 업데이트"는 조회 시점 기준이며 값 자체는 발표 주기를 따름을 UI에 명시
